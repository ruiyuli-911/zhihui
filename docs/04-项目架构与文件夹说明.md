# 智汇就业项目架构说明

## 当前仓库目标

当前版本优先解决三件事：

1. 前端目录与页面脚手架清晰可读。
2. 云函数目录可直接扩展与部署。
3. 仓库统一使用 UTF-8，避免后续继续出现乱码。

## 前端框架说明

- `app.js` 负责云开发初始化、系统信息初始化、登录状态恢复、更新检查。
- `utils/constants.js` 集中定义页面路由、角色、状态、云函数名称。
- `utils/page-factory.js` 提供统一页面工厂，所有页面先接入一致的生命周期和页面元信息。
- `components/framework-page/` 是当前统一占位框架组件，用于保证页面结构清晰、统一、可继续填充。

## 云函数框架说明

- `cloudfunctions/shared/response.js` 统一成功和失败返回结构。
- `cloudfunctions/shared/auth.js` 统一微信上下文读取。
- 每个业务云函数目录都带有 `index.js` 和 `package.json`，可以直接扩展业务逻辑。
- `account` 已保留实际基础能力，其余云函数先提供标准骨架。

## 编码规范

- 所有文本文件统一使用 UTF-8。
- `.editorconfig` 约束编辑器默认编码和换行。
- `.gitattributes` 约束 Git 工作区文本编码。
- `scripts/encoding-check.ps1` 可用于检查文本文件是否为有效 UTF-8。

## 后续填充原则

- 当前阶段先保证框架清晰，不调整页面业务内容。
- 页面逻辑统一从 `page-factory` 起步，避免再出现大量零散模板页。
- 云函数统一走 `shared/response` 返回格式，避免每个函数各写一套。
