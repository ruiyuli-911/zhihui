# 智汇小玉 — 系统处理逻辑（当前实现）

## 整体架构（当前运行版本）

```
微信小程序
  → POST /api/agent/chat  {user_id, session_id, message}
  → FastAPI app.py
    → ConversationOrchestrator.process()
      → IntentRouter.route_intent()
        → 强规则匹配: "报名第一个""确认""取消""查进度"
        → vivo AI 意图识别（受约束 JSON）: "推荐工作""再推荐一些""你是谁"
      → 服务层:
          ProfileService  — 查 accounts+jobseekers，脱敏手机号，身份证算年龄性别
          JobSearchService — 搜索岗位，年龄条件筛选(min_age/max_age)
          RecommendationService — 按偏好+收藏+浏览算分排序
          ApplicationService — prepare_application + submit_application（confirm_token）
          BehaviorService — 记录事件到 job_events + job_favorites
      → 返回结构化 ChatResponse {type, content, reply, actions}
  → 前端渲染（文本/岗位卡片/确认卡片/结果卡片）
```

## 一、消息处理流程

```
用户消息 → ConversationOrchestrator.process()

1. 获取/创建 session（内存中）
2. 加载用户资料到 profile（如果缺年龄）
3. IntentRouter.route_intent(message)
   → 先规则匹配
   → 规则不匹配则 vivo AI 识别（JSON）
   → 返回 VivoIntentResult {intent, slots, confidence}
4. 按状态机分发：
   IDLE → handle_idle
   SHOWING_JOBS → handle_showing_jobs
   WAITING_CONFIRM → handle_waiting_confirm
   COLLECTING_INFO → handle_collecting_info
5. 调用对应服务执行
6. 返回 ChatResponse
7. 保存到 history
```

### 意图路由 IntentRouter

**文件**: `server/orchestrator/intent_router.py`

```
第一关：强规则（不调 LLM）
  "报名第一个" → apply_job_by_index, slots={job_index: 1}
  "报名J123"   → apply_job_by_id, slots={job_id: "J123"}
  "确认"/"好"  → confirm_application
  "取消"/"不报名" → cancel_application
  "查报名进度"  → check_application_status
  "收藏"       → favorite_job

第二关：vivo AI 意图识别（受约束 JSON）
  输入: 用户自然语言
  输出: {"intent": "recommend_jobs", "slots": {"keyword": null, "city": null}, "confidence": 0.95}
  校验:
    - intent 在白名单内
    - job_index 是合法整数
    - confidence < 0.6 → 走 small_talk 兜底

第三关：兜底
  规则+LLM 都不匹配 → small_talk
```

## 二、推荐流程

```
用户说"推荐工作" / "找保安" / "再推荐一些"

1. IntentRouter 识别 intent:
   "推荐工作" → recommend_jobs
   "找保安"  → search_jobs {keyword: "保安"}
   "再推荐一些" → more_like_me

2. _do_recommend_or_search():
   a. 获取用户资料（年龄、偏好）
   b. 获取行为画像（收藏标签、浏览标签）
   c. JobSearchService.search_eligible_jobs():
      - 条件: auditStatus=approved, recruitStatus=recruiting
      - 年龄: (min_age IS NULL OR min_age <= age)
            AND (max_age IS NULL OR max_age >= age)
      - 排除已报名 + 已展示岗位
      - 关键词模糊匹配
   d. RecommendationService.score_and_sort_jobs():
      - 偏好匹配: +35
      - 收藏相似: +30
      - 多次浏览: +15
      - 地点匹配: +20
      - 薪资匹配: +15
      - 新发布: +10
      - 不喜欢: -50
   e. group_by_age_eligibility(): 按年龄合格/不限/不合格分组
   f. 取 top 10，保存到 session
   g. 返回 job_list 卡片
```

## 三、报名流程

```
用户选岗 → "报名第一个"

1. IntentRouter: apply_job_by_index {job_index: 1}
2. _do_select_job(1):
   - 从 session.last_jobs 锁定岗位
   - 设置 session.pending_job = job
   - 状态 → WAITING_CONFIRM
   - 返回 confirm_apply 卡片："XX岗位，确定报名吗？"

用户点"确认报名"

3. IntentRouter: confirm_application
4. _do_prepare_apply():
   - ApplicationService.prepare_application():
     a. get_job_detail 查岗位
     b. get_user_profile 查用户资料
     c. check_profile_completeness → 缺资料则一次问一个
     d. 幂等性检查（是否已报名）
     e. 生成 confirm_token（绑定 user+job+session，10分钟过期）
     f. 返回确认卡信息
   - 资料完整 → 返回 apply_confirm 卡片（含 confirm_token）
   - 缺资料 → 进入 COLLECTING_INFO

用户再次点"确认报名"（带 confirm_token）

5. 前端 submitApplication(confirmToken):
   POST /api/agent/confirm {user_id, confirm_token}
6. ApplicationService.submit_application():
   a. 校验 token（存在？未用？未过期？绑定当前用户+岗位？）
   b. 再次检查岗位状态
   c. 再次检查重复报名
   d. 写库 applications
   e. 返回 application_success
```

## 四、状态机（当前运行版本）

```
IDLE
  → recommend/search → SHOWING_JOBS
  → greeting/chat → IDLE（不改变状态）

SHOWING_JOBS
  → apply_job_by_index → WAITING_CONFIRM（选岗）
  → recommend/search → SHOWING_JOBS（重新搜索）
  → cancel → IDLE

WAITING_CONFIRM
  → confirm_application → SUBMITTING（prepare_apply）
    → 资料完整 → apply_confirm 卡片
    → 缺资料 → COLLECTING_INFO
  → cancel → IDLE
  → recommend/search → SHOWING_JOBS

COLLECTING_INFO
  → provide_info（用户提供信息）→ 收集到下一项或收齐
  → info_complete → WAITING_CONFIRM（回到准备报名）
  → cancel → IDLE

SUBMITTING
  → submit_application（带 confirm_token）→ SUCCESS
  → cancel → IDLE

SUCCESS
  → anything → IDLE
```

## 六、当前"机械"的地方和原因

| 现象 | 原因 |
|------|------|
| 推荐岗位回复用模板，语气生硬 | `_do_recommend_or_search` 直接返回 `"找到了 X 个岗位"` 模板，没有让 vivo AI 润色 |
| 报名确认卡片文字固定 | 确认卡文字是后端拼接的 `"您确认用以下信息报名吗？\n 姓名：..."`，没有让 AI 包装得更自然 |
| "再推荐一些"还是返回模板回复 | `more_like_me` 走的是同一套推荐逻辑，但回复还是模板 |
| 确认后的成功提示比较生硬 | `apply_success()` 模板返回固定文案 |

### 改进方向（让 vivo AI 润色关键节点的回复）

```
后端生成结构化数据（岗位列表、确认信息、结果）
  → 把结构化数据+上下文传给 vivo AI
  → vivo AI 润色语气（不改变业务内容）
  → 返回自然语言回复 + 结构化数据分开展示
```

例如目前是：
```
后端: "找到了 2 个岗位，按您的条件排好顺序了。"
```

可以改成：
```
后端生成 jobs 数据 → 传给 vivo AI
  → vivo AI 说: "我帮您看了下，目前有2个岗位比较适合您的情况。"
     （但 jobs 数组、score、reasons 仍然由后端控制）
```

目前 vivo AI 只在 IntentRouter 和 LLM闲聊两个场景用了。
推荐结果、确认卡文案、报名结果 这三个地方还没有让 AI 润色。
