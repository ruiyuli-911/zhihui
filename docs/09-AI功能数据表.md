## 一、系统架构

```
用户输入
   ↓
意图路由 (intent_detector.py)
   ↓
状态机 (state.py + employment_agent.py)
   ↓
规则分支 (employment_agent.py)
   ↓
工具调用 (tools.py) ──→ 数据库 (db_factory → db_client / mock)
   ↓
模板生成回复 (templates.py)
   ↓
结构化 JSON 返回 ──→ 前端渲染
```

## 二、意图路由表

| # | 意图 | 匹配条件 | 示例输入 | 关联操作 |
|---|------|---------|---------|---------|
| 1 | `search_job` | "找" + 岗位词 / "搜" / "查岗位" / "推荐" | 找保安工作、有保洁吗 | 调用 `search_jobs()` → 返回岗位列表 |
| 2 | `apply_job_by_id` | "报名" + 岗位编号 (J\d+) | 报名岗位 J1003、申请 J1001 | 调用 `get_job_detail()` 查岗位 → 进入确认 |
| 3 | `apply_job_by_index` | "报名第X个" | 报名第一个、报名第二个 | 从 `session.last_jobs` 取岗位 → 进入确认 |
| 4 | `confirm_apply` | 确认 / 确定 / 好的 / 嗯 | 确认、确定报名 | 检查资料完整性 → 提交报名 |
| 5 | `cancel_apply` | 取消 / 不要 / 算了 / 不报名 | 取消报名 | 清除待报名状态 |
| 6 | `application_status` | 报名进度 / 报名记录 / 我的报名 / 状态 | 查看我的报名、报名成功了吗 | 调用 `get_application_status()` |
| 7 | `job_detail` | 详情 / 看看 / 怎么样 / 介绍 | 岗位 J1003 详情、这个工作怎么样 | 调用 `get_job_detail()` |
| 8 | `interview_query` | 面试 | 我什么时候面试 | 返回引导提示（功能预留） |
| 9 | `provide_info` | 提供姓名/年龄/电话 | 我叫王建国、今年52岁、电话138... | 提取信息 → 存入 session → 收齐后报名 |
| 10 | `greeting` | 你好 / 您好 / 嗨 / 在吗 | 你好、在吗 | 返回问候语 |
| 11 | `small_talk` | 其他所有输入（不超过4字优先） | 哈哈、谢谢、无聊、今天天气好 | 返回闲聊模板回复 |
| 12 | `unknown` | 空输入 | -- | 返回引导提示 |

### 意图匹配优先级

```
apply_job_by_id  >  apply_job_by_index  >  search_job  >  job_detail  >
confirm_apply  >  cancel_apply  >  application_status  >  interview_query  >
greeting  >  small_talk  >  provide_info  >  unknown
```

## 三、状态机

### 3.1 状态定义

| 状态 | 含义 | 说明 |
|------|------|------|
| `IDLE` | 初始/空闲 | 等待用户输入，无上下文 |
| `SHOWING_JOBS` | 展示了岗位列表 | 等待用户选择岗位或重新搜索 |
| `WAITING_CONFIRM` | 等待确认报名 | 已选中岗位，等待用户确认/取消 |
| `COLLECTING_INFO` | 收集资料 | 缺少姓名/电话/年龄，逐项追问 |
| `SUBMITTING` | 提交报名 | 正在调用 `apply_job()` |
| `SUCCESS` | 报名完成 | 展示结果，可继续搜索 |

### 3.2 状态转移

                    │                              │

 │      │         │  JOBS   │           │                │
 └──────┘         └──────────┘           └───────┬────────┘
     ▲              ▲     ▲                      │
     │              │     │         确认          │
     │              │     └───────────────────────┤
     │              │                            │
     │              │  重新搜索                   ▼
     │              │                     ┌──────────────────┐

## 四、工具列表

### 4.1 核心工具

| 工具名 | 文件 | 参数 | 返回 | 调用时机 |
|--------|------|------|------|---------|
| `search_jobs()` | tools.py | keyword, min_salary, distance_km, provide_food, provide_housing | `{total, jobs: [{job_id, title, company_name, salary, location, benefits}]}` | 用户搜索岗位时 |
| `get_job_detail()` | tools.py | job_id | `{job: {...}, error}` | 查看详情或按ID报名时 |
| `apply_job()` | tools.py | job_id, user_id, user_name, user_phone, user_age | `{success, need_info, missing_fields, error}` | 用户确认报名后 |
| `get_application_status()` | tools.py | user_id | `{total, records: [{job_title, status_text, ...}]}` | 用户查进度时 |
| `get_user_profile()` | tools.py | user_id | `{profile: {name, phone, age}}` | 报名前检查资料完整性 |

### 4.2 工具数据流

```
用户说"找保安工作"
  → search_jobs(keyword="保安")
  → db.query("jobs", where={auditStatus, recruitStatus, title})
  → 返回 [{job_id, title, salary, benefits...}]
  → 模板格式化 → {type: "job_list", jobs: [...], reply: "..."}

用户说"报名岗位 J1003"
  → get_job_detail(job_id="J1003")
  → db.get("jobs", "J1003")
  → 返回岗位信息
  → 模板格式化 → {type: "confirm_apply", job: {...}, reply: "..."}

用户说"确认" + 资料完整
  → apply_job(job_id, user_id, user_name, user_phone, user_age)
  → db.add("applications", {...}) + db.update("jobs", ..., {applyCount:+1})
  → 返回报名结果
  → 模板格式化 → {type: "result", status: "success", ...}
```

## 五、前端消息类型

| type | 含义 | 前端渲染 | 数据内容 |
|------|------|---------|---------|
| `job_list` | 岗位卡片列表 | 岗位卡片（标题/薪资/公司/福利标签）+ 报名按钮 | `{summary, jobs: [{job_id, title, company_name, salary, location, benefits}]}` |
| `confirm_apply` | 确认报名 | 岗位信息卡片 + 确认/取消按钮 | `{text, job: {title, salary, benefits}, actions}` |
| `collect_info` | 收集资料 | 缺哪些字段 + 示例说法 | `{text, missing_fields: ["姓名",...], collected}` |
| `result` | 操作结果 | 成功/失败图标 + 标题 + 描述 + 操作按钮 | `{status: "success"|"error", title, description}` |
| `job_detail` | 岗位详情 | 完整岗位信息卡片 + 报名按钮 | `{job: {title, salary, company_name, requirement, ...}}` |
| `text` | 纯文本 | 普通消息气泡 | `{text}` |
| `error` | 错误提示 | 红色警示条 | `{text}` |

## 六、会话数据结构

```python
session = {
    "user_id": str,              # 用户 openid
    "state": State,              # 当前对话状态
    "history": [],               # 对话历史 [{role, content}]
    "last_jobs": [],             # 上次搜索的岗位列表 [{job_id, title, salary, ...}]
    "pending_job": {},           # 待报名的岗位
    "collecting_fields": [],     # 还在收的字段 ["姓名", "联系电话", "年龄"]
    "collected_info": {},        # 已收的信息 {"name": "王建国", "phone": "...", "age": 52}
    "last_result": {},           # 上次操作结果
    "location": {},              # 用户位置
}
```

## 九、核心原则

| 原则 | 说明 |
|------|------|
| **不编造数据** | 岗位信息、报名结果、面试时间必须来自工具返回的真实数据 |
| **确认后再执行** | 报名等写操作，必须先让用户确认 |
| **状态可控** | 对话由状态机控制，不跳过必要步骤 |
| **闲聊不办事** | 闲聊只回复不执行业务，不丢上下文 |
| **适老表达** | 口语化、一次只说一件事、明确告诉用户下一步做什么 |
